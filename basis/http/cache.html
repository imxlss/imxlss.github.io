# HTTP 缓存机制

缓存是指代理服务器或客户端本地磁盘内保存的资源副本。

按照缓存策略分为强制缓存和协商缓存，分别利用了不同的 Http 首部字段。

# 强制缓存

强制缓存表示不需要向原服务器请求资源。

涉及到的首部字段：

Expires（http 1.0）：资源失效的日期

Cache-Control（http 1.1）：通过设置指令控制缓存的行为，优先级高于 Expires。

常用指令：

  • max-age：表示资源保存为缓存的最长时间
  • public：表示其他用户也可利用缓存
  • private：只有特定用户可利用缓存
  • no-cache：防止从缓存中返回过期的资源。使用前要向服务器进行确认。
  • no-store：不进行缓存

# 协商缓存

协商缓存表示使用缓存前先向服务器确认资源是否发生了变动。

涉及到的首部字段：

Last-Modified（http 1.0）：资源上次修改的时间

If-Modified-Since：请求首部字段；比较资源的更新时间

ETag（http 1.1）：实体的唯一标识

If-None-Match：请求首部字段；比较ETag，不一致时处理

If-Macth：与If-None-Match作用相反
