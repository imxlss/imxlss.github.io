# HTTP 访问控制（CORS）

跨域资源共享（CORS）是一种机制，它使用额外的 HTTP 头来告诉浏览器，让运行在一个 origin(domain)上的 Web 应用被准许访问来自不同源服务器上的指定的资源。

# 跨域是什么？

当一个页面从 与该页面本身所在的服务器有不同的域名、协议或端口的服务器上 请求一个资源时，就会发起一个跨域请求。

浏览器的同源策略：出于安全原因，浏览器限制从脚本内发起的跨域 HTTP 请求。

TIP

不一定是浏览器限制了跨域请求的发起，也可能是跨域请求正常发起了，但浏览器拦截了返回结果。

# HTTP 响应首部字段

  • Access-Control-Allow-Origin
Access-Control-Allow-Origin: <origin> | *
1

origin参数的值指定了允许访问该资源的外域 URL。对于不需要携带身份凭证的请求，服务器可以指定该字段的值为通配符，表示允许来自所有域的请求。

  • Access-Control-Allow-Credentials
Access-Control-Allow-Credentials: true
1

Access-Control-Allow-Credentials指定了当浏览器的credentials设置为 true 时，是否允许浏览器读取response的内容。

  • Access-Control-Expose-Headers
Access-Control-Expose-Headers: X-My-Custom-Header, X-Another-Custom-Header
1

跨域访问时，XMLHttpRequest对象的getResponseHeader()方法只能拿到最基本的响应头。如果要访问其他的响应头，则需要服务器设置本响应头。

  • Access-Control-Max-Age
Access-Control-Max-Age: <delta-seconds>
1

用于预检请求。指定了预检请求的结果多长时间内有效。

  • Access-Control-Allow-Methods
Access-Control-Allow-Methods: <method>, ...
1

用于预检请求。指明了实际请求所允许使用的 HTTP 方法。

  • Access-Control-Allow-Headers
Access-Control-Allow-Headers: <field-name>, ...
1

用于预检请求。指明了实际请求中允许携带的首部字段。

# HTTP 请求首部字段

TIP

当使用XMLHttpRequest对象发起跨域请求时，以下首部字段会被自动设置。

  • Origin
Origin: <origin>
1

origin参数值为源站 URI。

  • Access-Control-Request-Method
Access-Control-Request-Method: <method>
1

用于预检请求。把实际请求要用的 HTTP 方法告诉服务器。

  • Access-Control-Request-Headers
Access-Control-Request-Headers: <field-name>, ...
1

用于预检请求。将实际请求要带的首部字段告诉服务器。
