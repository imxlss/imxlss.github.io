# 事件绑定和处理

在小程序内，除了标准 HTML5 中遇见的 touchstart 等事件外，增加了 tap 类的事件，主要包括以下几种：

事件名称 说明
tap 手指触摸后马上离开
longpress 手指触摸后，超过 350ms 后再离开
longtap 同 longpress(推荐使用 longpress)

事件的绑定通过bind*属性来实现。

例：绑定一个tap事件，事件处理函数为indexDetail

<view bindtap="indexDetail"></view>
1

# 事件冒泡

小程序内的事件分为可冒泡和不可冒泡的事件，除了 submit、input 之类的事件，多数是可冒泡的事件

对于事件的绑定，除了bind*，还可以catch*:

  • bind不会阻止冒泡，变形写法为bind:*
  • catch会阻止事件继续冒泡，变形写法为catch:*

# 事件捕获

捕获先于冒泡的触发，绑定捕获事件，可以使用capture-bind，capture-catch。capture-catch将中断捕获阶段和取消冒泡阶段。

先后调用 handleTap2、handleTap4、handleTap3、handleTap1。

<view
  id="outer"
  bind:touchstart="handleTap1"
  capture-bind:touchstart="handleTap2"
>
  outer view
  <view
    id="inner"
    bind:touchstart="handleTap3"
    capture-bind:touchstart="handleTap4"
  >
    inner view
  </view>
</view>

# 事件对象

当事件触发时，处理函数会响应，传入event对象，通过event对象可以获取事件触发时的一些信息，包括时间戳、detail 等。

传参需要在标签上增加data-xx这样的属性，然后可以在event对象的target/currentTarget的dataset获取参数。

正确获取dataset的姿势是使用currentTarget

  • target: 触发事件的源组件
  • currentTarget: 事件绑定的当前组件

# 路由

小程序提供了对应 a 标签和 location 对象的方法：navigator 组件和 wx 中的导航相关函数

路由栈

在小程序中，路由是由路由栈来维护的，小程序的路由栈中最多维护 5 个页面

小程序中跳转页面可以通过下面两种方式：

  1. 使用 navigator 组件：
<navigator url="跳转页面URL">跳转到新页面</navigator>
1
  1. 使用 wx 中的导航相关函数：
<view bindtap="gotoUrl">跳转页面</view>
1
Page({
  gotoUrl() {
    let url = 'pages/another/url';
    wx.navigateTo({
      url
    });
  }
});
