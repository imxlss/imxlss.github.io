# async && await

# async

async 起什么作用

asyn函数保证返回值是一个 Promise 对象。

如果在 async 函数内return一个直接量，async 会使用Promise.resolve()将其封装成一个 Promise 对象。

Promise 对象的特点——无等待。所有在没有await的情况下执行async函数，它会立即执行，返回一个Promise对象，绝对不会阻塞后面的语句。

# await

await 在等什么

await等的是一个返回值。

  1. 如果await等到的是一个 Promise 对象，它会阻塞后面的代码，等待Promise对象resolve，然后得到resolve后的值，作为await表达式的运算结果。

  2. 如果await等到的不是一个Promise对象，那么await表达式的运算结果就是它等到的消息。

# async/await 的优势

async/await优势在于处理then链。

TIP

await 只能出现在 async 函数中

# proxy

ES6 原生提供了 Proxy 构造函数，用来生成 Proxy 实例。

var proxy = new Proxy(target, handler);
1

new Proxy()表示生成一个Proxy实例，target参数表示所要拦截的目标对象，handler参数也是一个对象，用来定制拦截行为。

# 例子

var proxy = new Proxy(
  {},
  {
    get: function(target, property) {
      return 35;
    }
  }
);

proxy.name; // 35
proxy.age; // 35
proxy.title; // 35

Proxy接受两个参数。

# Reflect

将 Object 对象的一些明显属于语言内部的方法（比如 Object.defineProperty），放到 Reflect 对象上。

Reflect 对象上的方法和Proxy对象的方法一一对应，只要是Proxy对象的方法，就能在Reflect对象上找到对应的方法。

可以通过调用对应的Reflect方法，完成默认行为。

# static

ES6 中的类（class）相当于实例的原型，所有在类中定义的方法，都会被实例继承。如果在一个方法前，加上static关键字，就表示该方法不会被实例继承，而是直接通过类来调用，这就称为静态方法。

TIP

静态方法只能通过类来调用

# 函数柯里化

currying 又称部分求值。

一个currying的函数首先会接受一些参数，接受了这些参数之后，该函数并不会立即求值，而是继续返回另外一个函数，刚才传入的参数在函数形成的闭包中被保存起来。待到函数被真正需要求值的时候，之前传入的所有参数都会被一次性用于求值。

下面函数说明了currying的思想：

let cost = (function() {
  let args = []; // 用来保存参数

  return function() {
    if (arguments.length === 0) {
      let money = 0;
      for (let i = 0, l = args.length; i < l; i++) {
        money += args[i];
      }
      return money;
    } else {
      // [].push.apply(args, arguments);
      args.push(...arguments);
    }
  };
})();

cost(100); // 未真正求值
cost(200); // 未真正求值
cost(300); // 未真正求值

console.log(cost()); // 求值并输出：600

# 一个通用的 function currying（）{}

let currying = function(fn) {
  let args = [];

  return function() {
    if (arguments.length === 0) {
      return fn.apply(this, args);
    } else {
      args.push(...arguments);
      return arguments.callee; // 返回另外一个函数
    }
  };
};

let fn = (function() {
  let money = 0;

  return function() {
    for (let i = 0, l = arguments.length; i < l; i++) {
      money += arguments[i];
    }

    return money;
  };
})();

let cost = currying(fn);

cost(100); // 未真正求值
cost(200); // 未真正求值
cost(300); // 未真正求值

console.log(cost()); // 求值并输出：600

# 反柯里化(uncurrying)

柯里化是预先填入一些参数；反柯里化是把原来已经固定的参数或者 this 上下文等当做参数延迟到未来传递。

TIP

把this.method(arg1, arg2 ...)转化为method(this, arg1, arg2 ...)以实现方法借用和 this 的泛化。

反柯里化使本来只有特定对象才适用的方法，扩展到更多对象。

uncurrying 的实现之一：

Function.prototype.uncurrying = function() {
  let self = this;

  return function() {
    let obj = Array.prototype.shift.call(arguments);
    return self.apply(obj, arguments);
  };
};

# 函数声明和函数表达式

函数声明：使用function关键字声明函数

function name([param1]) {
  // statements
}
1
2
3

函数表达式：将函数赋值给一个变量

let function_expression = function [name]([param1]) {
   // statements
};
1
2
3

两者区别：

  • 函数声明必须有name，函数表达式可省略
  • 表达式里的name不能在函数外面用，只能在函数内部用
  • 函数声明会被预解析，函数表达式不会

TIP

IIFE 中的函数是函数表达式

# 立即执行函数

# 立即执行函数是什么

立即执行函数就是：

  1. 声明一个匿名函数
  2. 马上调用这个匿名函数

用另一对括号把匿名函数包起来是为了兼容 JS 的语法；不加会报错

# 立即执行函数有什么用

只有一个作用：创建一个独立的作用域。 这个作用域里面的变量，外面访问不到（即避免变量污染）。

TIP

可以来解决for循环中使用var的面试题

TIP

IFEE 相当于创建一个匿名函数并执行，当函数被理解成表达式时就会执行，如+(function(){})()。
