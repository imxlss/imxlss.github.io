# new 操作符

# new 的作用

  • 返回一个对象
  • 该对象通过原型链和构造函数的原型属性联系起来
  • 返回值为对象时可以正常返回，否则返回this

# 实现一个 new 操作符

function myNew(Constructor, ...args) {
  let obj = Object.create(Constructor.prototype);
  let result = Constructor.call(obj, ...args);
  return result instanceof Object ? result : obj;
}

# instanceof 的实现

instanceof 判断的原理是：右边变量的prototype对象是否在左边变量的原型链上。

TIP

简单说，原型链就是__proto__组成的链。

function myInstanceof(left, right) {
  let proto = left.__proto__;
  let prototype = right.prototype;
  while (true) {
    if (proto === null) {
      return false;
    }
    if (proto === prototype) {
      return true;
    }
    proto = proto.__proto__;
  }
}

# 关于垃圾回收

JavaScript 具有垃圾回收机制。

自动垃圾回收就是找出那些不再使用的值，释放其占用的内存。

# 引用计数垃圾收集

最初的垃圾回收算法。

如果没有其他对象指向一个对象了，说明该对象已经不再被需要了。

该算法存在循环引用的问题。

# 标记清除算法

标记清除算法将不再使用的对象定义为无法到达的对象。简单来说，就是从根部(js 中就是全局对象)出发，定时扫描内存中的对象。凡是能从根部到达的对象，都是还需要使用的。那些无法由根部出发触及到的对象被标记为不再使用，稍后进行回收。

工作流程：

  • 垃圾收集器会在运行的时候会给存储在内存中的所有变量都加上标记。
  • 从根部出发将能触及到的对象的标记清除。
  • 那些还存在标记的变量被视为准备删除的变量。
  • 最后垃圾收集器会执行最后一步内存清除的工作，销毁那些带标记的值并回收它们所占用的内存空间。

# 内存泄漏

内存泄漏的识别方法

经验法则是，如果连续五次垃圾回收之后，内存占用一次比一次大，就有内存泄漏。 这就要求实时查看内存的占用情况。
