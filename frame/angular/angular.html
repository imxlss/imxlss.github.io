# ElementRef 的应用

可以使用依赖注入 ElementRef 的方式，轻松获取到封装后的 native 元素

直接注入ElementRef, this.elementRef为组件本身 即下例中的<my-app></my-app>

import { Component, ElementRef, AfterViewInit } from '@angular/core';

@Component({
  selector: 'my-app',
  template: `
    <h1>Welcome to Angular World</h1>
    <div>Hello {{ name }}</div>
  `
})
export class AppComponent {
  name: string = 'Semlinker';

  // 在构造函数中 this.elementRef = elementRef 是可选的，编译时会自动赋值
  // function AppComponent(elementRef) { this.elementRef = elementRef; }
  constructor(private elementRef: ElementRef) {}

  ngAfterViewInit() {
    // 模板中的元素已创建完成
    console.dir(this.elementRef.nativeElement.querySelector('div'));
    // let greetDiv: HTMLElement = this.elementRef.nativeElement.querySelector('div');
    // greetDiv.style.backgroundColor = 'red';
  }
}

# 优化

可以使用 @ViewChild() 获取元素。使用 renderer2对象 提供的 API 优雅地给 native 元素设置属性和样式。

TIP

要尽量减少应用层与渲染层之间强耦合关系，从而让应用能够灵活地运行在不同环境。

# ViewChild

属性装饰器，用于配置一个视图查询。 变更检测器会在视图的 DOM 中查找能匹配上该选择器的第一个元素或指令。 如果视图的 DOM 发生了变化，出现了匹配该选择器的新的子节点，该属性就会被更新。

TIP

在 ngAfterViewInit 钩子函数中会设置这些视图查询。

// 创建了一个指令，它会对具有 ngModel 指令的 DOM 元素设置 valid 和 invalid 属性。

@Directive({ selector: '[ngModel]' })
class NgModelStatus {
  constructor(public control: NgModel) {}
  @HostBinding('class.valid') get valid() {
    return this.control.valid;
  }
  @HostBinding('class.invalid') get invalid() {
    return this.control.invalid;
  }
}

@Component({
  selector: 'app',
  template: `
    <input [(ngModel)]="prop" />
  `
})
class App {
  prop;
}

// 创建一个指令，为宿主元素添加role='button'属性；
// 同时根据是否按下判断 pressed 类是否生效

@Directive({ selector: '[buttonPress]' })
export class ButtonPressDirective {
  @HostBinding('attr.role') role = 'button';
  @HostBinding('class.pressed') isPressed: boolean;

  @HostListener('mousedown') hasPressed() {
    this.isPressed = true;
  }

  @HostListener('mouseup') hasReleased() {
    this.isPressed = false;
  }

  constructor() {}
}

# 依赖注入（DI）

依赖，是当类需要执行其功能时，所需要的服务或对象。

DI，是一种编码模式，其中的类会从外部源中请求获取依赖，而不是自己创建它们。

@Injectable() 是每个 Angular 服务定义中的基本要素。

TIP

实际上，@Injectable() 对所有服务都是必须的。

# 配置服务提供商

# 服务提供商

一个实现了Provider接口的对象。定义了如何获取与 DI 令牌（token） 相关联的可注入依赖。 注入器会使用这个提供商来创建它所依赖的那些类的实例。

TIP

提供商会告诉注入器如何创建服务

# 如何配置服务提供商

可以在三种位置之一设置元数据，以便在应用的不同层级使用提供商来配置注入器：

  • 在服务本身的@Injectable()装饰器中 —— providerIn
  • 在 NgModule 的 @NgModule() 装饰器中 —— providers
  • 在组件的 @Component() 装饰器中 —— providers

# 依赖注入令牌

在 Angular 中，DI 令牌用于在依赖注入系统中查找服务提供商。

注入器维护一个内部令牌-提供商的映射表，当请求一个依赖项时就会引用它。令牌就是这个映射表的键。

# 参数装饰器

  • @Optional() —— 该依赖是可选的
  • @Host() —— 禁止在宿主组件以上的搜索
  • @Inject() —— 指定自定义提供商
  • @Self() —— 注入器只在该组件的注入器中查找提供商
  • @SkipSelf() —— 跳过局部注入器，在注入器树中向上查找，以发现哪个提供商满足该依赖
